PILLAR I: DEVELOPMENT QUALITY

Specification Driven Development

The first step to implement a feature is not to start coding it, but to write whatever has to be implemented in a clear and precise specification document.

Planning ("discussion" with AI around feature)

Document (Write task detail)

Generate (Unit tests and stub file)

Implement (Code the functionality)

Verify (Run tests)

Deploy

Audit

ADR : Architectural Decision Record

Principles which every piece of code should adhere to. E.G. "no PII in any code file", "no hardcoded secrets".

PRD : Product Requirement Document

One per project. It should contain purpose, features, functionality and behavior of the project. What and Why.

SRS : Software Requirement Specification

Functional and non functional details. The how of project or even module. Something like Jira epic.

feature.md : Definition of feature in that directory

This would be something like Jira story.
PILLAR II: Robust and Secure

Using Gen AI capabilites

Ensuring that code is secure and robust, it doesn't crash easily and is resilient to deviations from "happy path".

Robust

HOW : After document the feature, ask AI to generate "exhaustive and comprehensive unit tests".

Secure

HOW : Either everyone should have the standard guidelines (ADR, PCI-DSS, ISO), or they should exist in a centralized location (vertex or github) which helps in unit test generation.
